# 📰 全球新闻简报 — News Briefing Skill v2.1

## 功能概述

自动抓取全球重要新闻，按 **政治 / 经济 / 军事 / 科技 / 社会** 五大分类整理，经过以下处理后推送到飞书：

1. **自动翻译** — 海外英文信源自动翻译为中文（通过 LLM）
2. **交叉核实** — 跨源相似报道聚类，2+源报道标记为已核实，显示其他来源
3. **存疑标注** — 单源低可信度、敏感词、标题夸张等自动标注⚠️
4. **去重评分** — MD5指纹 + 标题相似度去重，多维评分排序

## 评分逻辑

每条新闻基础分 50，根据以下因素加减分：

| 因素 | 分值 | 说明 |
|------|------|------|
| Tier 1 信源 | +15 | Reuters/AP/BBC/新华社等顶级信源 |
| Tier 2 信源 | +8 | Al Jazeera/财新/WSJ 等 |
| 关键词匹配 | +10×命中数 | 根据分类关键词匹配（上限30） |
| 中国相关 | +10 | 标题/摘要包含中国关键词 |
| 突发新闻 | +20 | 包含"突发/breaking/快讯"等标识 |
| 3+源交叉核实 | +15 | 多个独立源报道同一事件 |
| 2源交叉核实 | +8 | 两个独立源报道 |
| 存疑标记 | -10 | 单源/敏感词/标题夸张等 |
| 有摘要 | +5 | 摘要长度>20字 |
| 标题过短 | -15 | 标题<10字 |
| 标题过长 | -5 | 标题>100字 |

## 交叉核实逻辑

去重后对所有新闻进行相似度聚类（阈值 0.5）：
- **3+ 独立源**：✅ 已核实（多源确认）— 加15分
- **2 独立源**：✅ 已交叉验证 — 加8分
- **单源**：无标记，但不加分

推送时会显示其他报道该新闻的来源列表。

## 存疑标注逻辑

以下条件触发存疑标注（⚠️）：
1. 单源 + Tier 3 信源（热搜聚合等未经验证的来源）
2. 标题含煽动性/敏感词（"揭秘/内幕/shocking/unconfirmed"等）
3. 标题过于夸张（过多感叹号、全大写英文）
4. 信息不完整（无摘要且评分较低）

存疑项会被降权10分，并在推送中显示具体原因。

## 去重策略

1. **指纹去重**：相同标题+来源生成 MD5 指纹
2. **相似度去重**：标题相似度 > 0.6 的只保留分高的
3. **缓存过滤**：已推送过的 48 小时内不再推送

## 配置说明

配置文件 `config.json`：

- **delivery**: 飞书推送目标配置
- **rss_feeds**: 按类别分组的 RSS 源列表
  - `tier`: 信源权重（1=顶级，2=优质，3=聚合）
  - `lang`: 语言标识（zh/en），用于显示国旗标识
- **categories**: 五大分类的关键词配置
- **scoring**: 评分参数和输出限制

## 添加新信源

在 `config.json` 的 `rss_feeds` 对应分类中添加：
```json
{"name": "新源名称", "url": "RSS地址", "lang": "zh", "tier": 2}
```

## 注意事项

- 部分 RSS 源可能因网络原因抓取失败，不影响整体输出
- 热搜聚合源（微博/知乎/Twitter）可能不稳定，RSS 源是主要信源
- 缓存文件 `news_cache.json` 自动维护，48小时过期清理
- 所有抓取并发执行，单次运行约 30-60 秒
